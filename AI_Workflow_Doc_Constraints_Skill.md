# 模型分工约束

## 适用范围

本约束适用于本目录下后续与分析、代码实现相关的协作过程。

## 分工规则

- 所有分析类问题默认使用 `gpt-5.4`。
- 各种代码任务默认使用 `gpt-5.3-codex`。
- 若用户后续明确指定其他模型，以用户最新要求为准。

## 执行说明

- “分析类问题”包括架构权衡、风险识别、边界条件讨论、脚本行为解读、故障定位与方案比较等。
- “代码任务”包括 RTL/脚本/工具代码的新实现、重构、接口调整、命名/风格修正与测试修复等。

## 协作流程约定

- 每次改代码或脚本之前，先询问用户是否还有其他需求；只有用户明确表示“可以开始写代码”后再开始修改。
- 修改完成后先在本地完成必要检查和验证。
- 修改完成后不要自动上传文件；必须先询问用户是否需要上传，等用户明确确认后再执行上传。


# Markdown 文档编写约束

## 适用范围

本约束适用于本目录下后续新增或更新的 Markdown 文档。

## 语言约束

- 默认使用中文（简体）编写全文。
- 除非用户明确要求英文或中英双语，否则不使用英文作为主语言。
- 代码、命令、寄存器名、信号名、路径名保持原文，不做翻译。
- 必要英文术语可保留，但需尽量配中文说明。

## 表达约束

- 标题与正文使用中文语义，避免中英文混杂造成歧义。
- 结论、风险、限制条件优先明确写出，避免模糊描述。
- 涉及数值、位宽、分辨率、单位时，必须写清楚单位与前提。

## 执行约定

- 从本文件创建后开始生效。
- 若后续用户给出新的文档语言要求，以用户最新要求为准。
